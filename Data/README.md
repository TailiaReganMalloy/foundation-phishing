# Data Directory Overview

This document summarizes the five data subfolders, the files they contain, and the column schema for the CSV exports. The JSON and MAT files in each folder are serialized versions of the same schema.

## Emails/
- **Files**: `Emails.csv`, `Emails.json`, `Emails.mat`
- **Schema**:
  | Column | Description |
  | --- | --- |
  | `EmailId` | Unique identifier for the rendered email stimulus. |
  | `BaseEmailID` | Identifier for the underlying base prompt or template. |
  | `Author` | Indicates whether the email text was written by a human or generated by GPT. |
  | `Style` | Formatting style applied to the email (e.g., Plaintext, GPT). |
  | `Type` | Labeled email class such as `phishing` or `ham`. |
  | `Sender Style` | High-level template or brand associated with the sender persona. |
  | `Sender` | From-address displayed to the participant. |
  | `Subject` | Email subject line. |
  | `Sender Mismatch` | Binary flag (0/1) for mismatched sender identity cues. |
  | `Request Credentials` | Binary flag for explicit credential requests. |
  | `Subject Suspicious` | Binary flag noting if the subject contains suspicious language. |
  | `Urgent` | Binary flag for urgent tone. |
  | `Offer` | Binary flag for the presence of an offer/incentive. |
  | `Link Mismatch` | Binary flag indicating hyperlink text does not match the destination. |
  | `Prompt` | Prompt text supplied to GPT (blank for human-authored emails). |
  | `Body` | Full HTML or plaintext body delivered to participants. |

## Embeddings/
- **Files**: `Embeddings.csv`, `Embeddings.json`, `Embeddings.mat`
- **Schema**:
  | Column | Description |
  | --- | --- |
  | `EmailId` | Identifier linking to `Emails.EmailId`. |
  | `BaseEmailID` | Underlying base email template ID. |
  | `Author` | Human or GPT authorship flag. |
  | `Style` | Rendering style label. |
  | `Embedding` | Vector (stored as a serialized list) representing the semantic embedding of the email body. |
  | `Phishing Similarity` | Similarity score between the embedding and a phishing centroid. |
  | `Ham Similarity` | Similarity score between the embedding and a ham centroid. |

## Participant/
- **Files**: `ParticipantData.csv`, `ParticipantData.json`, `ParticipantData.mat`
- **Schema**:
  | Column | Description |
  | --- | --- |
  | `Unnamed: 0` | Saved index column from the export. |
  | `UserId` | Internal identifier for the participant. |
  | `Experiment` | Experiment number the participant completed. |
  | `Condition` | Experimental condition label (e.g., Human Written LLM Styled). |
  | `PhaseValue` | Countdown of trials remaining within the current phase. |
  | `PhaseTrial` | Trial number within the current phase. |
  | `ExperimentTrial` | Global trial number within the experiment. |
  | `DataType` | Record type (e.g., `Response`). |
  | `EmailId` | Email stimulus presented on the trial. |
  | `EmailType` | Stimulus label (`phishing` or `ham`). |
  | `Decision` | Participant judgment (`true` for phishing detected, `false` otherwise). |
  | `Confidence` | Confidence rating scale (integer). |
  | `EmailAction` | Action option chosen (integer-coded). |
  | `ReactionTime` | Response latency in milliseconds. |
  | `Correct` | Accuracy flag comparing decision to ground truth. |
  | `MessageNum` | Sequential message number within chatbot interactions. |
  | `Message` | Free-text chatbot message content. |
  | `Age` | Self-reported age. |
  | `Gender` | Self-reported gender code. |
  | `Education` | Self-reported education code. |
  | `Country` | Self-reported country. |
  | `Victim` | Self-reported prior victimization status. |
  | `Chatbot` | Chatbot group assignment or notes. |
  | `Q0` – `Q5` | Numeric ratings collected during the main questionnaire block. |
  | `PQ0` – `PQ5` | Post-questionnaire free-text rationales corresponding to `Q0`–`Q5`. |

## Questions/
- **Files**: `Questions.csv`, `Questions.json`, `Questions.mat`
- **Schema**:
  | Column | Description |
  | --- | --- |
  | `Unnamed: 0` | Saved index column from the export. |
  | `MturkId` | Mechanical Turk worker identifier. |
  | `Age` | Self-reported age. |
  | `Gender` | Self-reported gender code. |
  | `Education` | Self-reported education code. |
  | `Country` | Self-reported country. |
  | `Victim` | Self-reported phishing victimization status. |
  | `Consent` | Consent flag captured at survey start. |
  | `Q0` – `Q5` | Likert-style questionnaire responses during the study. |
  | `Userid` | Internal user identifier linking to participant records. |
  | `PQ0` – `PQ5` | Free-text rationales for the corresponding Likert responses. |
  | `Rejected` | Binary flag showing whether the session was rejected. |

## Responses/
- **Files**: `Responses.csv`, `Responses.json`, `Responses.mat`
- **Schema**:
  | Column | Description |
  | --- | --- |
  | `Unnamed: 0` | Saved index column from the export. |
  | `MturkId` | Worker identifier. |
  | `Userid` | Internal user identifier. |
  | `EmailId` | Identifier of the email presented on the trial. |
  | `TrainingPeriod` | Phase label (`preTraining`, `Training`, `postTraining`, etc.). |
  | `Trial` | Trial number within the phase. |
  | `Condition` | Experimental condition label for the trial. |
  | `Decision` | Participant classification of the email (True/False for phishing). |
  | `EmailType` | Ground-truth label (`phishing` or `ham`). |
  | `Action` | Button/action selected (integer-coded). |
  | `Confidence` | Confidence rating. |
  | `ReactionTime` | Response latency in milliseconds. |
  | `Correct` | Accuracy flag for the trial. |
